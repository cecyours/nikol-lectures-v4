CH01_S04_Forms_And_Validation

HTML forms collect user input and submit data. HTML5 provides helpful built-in validation using attributes like required, min/max, minlength, and pattern.

Live form (try submitting)

Required field.
Must be a valid email format.
6–12 characters.

Only letters, minimum 5 characters.

Code (same idea)

<form action="register.php" method="post">
  <label for="name">Name:</label>
  <input id="name" name="name" required />

  <label for="email">Email:</label>
  <input type="email" id="email" name="email" required />

  <label for="password">Password:</label>
  <input type="password" id="password" name="password"
         minlength="6" maxlength="12" required />

  <label for="age">Age:</label>
  <input type="number" id="age" name="age" min="18" max="60" required />

  <input type="submit" value="Register" />
</form>

Important note

HTML5 validation is client-side only and can be bypassed. In a MERN app, you must also validate on the server (Node/Express) and database.
